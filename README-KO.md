## ⚡ 개요

**MiroFish**는 다중 에이전트 기술로 구동되는 차세대 AI 예측 엔진입니다. 현실 세계의 시드 정보(뉴스, 정책 초안, 금융 신호 등)를 추출하여 고충실도의 평행 디지털 월드를 자동으로 구축합니다. 이 공간에서 수천 개의 독립적인 인격, 장기 기억, 행동 논리를 가진 지능형 에이전트들이 자유롭게 상호작용하며 사회적 진화를 거듭합니다. 당신은 "신의 시점"에서 변수를 동적으로 주입하여 미래 궤적을 정밀하게 추론할 수 있습니다 — **디지털 샌드박스에서 미래를 리허설하고, 무수한 시뮬레이션 후 결정에서 승리하세요**.

> 당신은 필요한 것: 시드 자료 업로드(데이터 분석 보고서나 흥미로운 소설 이야기) + 자연어로 예측 요구사항 설명</br>
> MiroFish가 반환: 상세한 예측 보고서 + 깊이 상호작용 가능한 고충실도 디지털 월드

### 우리의 비전

MiroFish는 현실을 매핑하는 군집 지능 거울을 만드는 데 전념합니다. 개별 상호작용으로 촉발된 집단적 출현(emergence)을 포착하여 전통적인 예측의 한계를 극복합니다:

- **거시적 수준에서**: 의사결정자를 위한 리허설 연구실로, 정책과 홍보를 zero-risk로 테스트
- **미시적 수준에서**: 개인 사용자를 위한 창의적 샌드박스로 — 소설 결말 추론이든 상상 시나리오 탐색이든, 모든 것이 재미있고 쉽게 접근 가능

진지한 예측부터 재미있는 시뮬레이션까지, 우리는 모든 "만약에"가 결과를 볼 수 있게 만들어, 만물 예측을 가능하게 합니다.

## 🔄 워크플로우

1. **그래프 구축**: 시드 추출 & 개별/집단 기억 주입 & GraphRAG 구축
2. **환경 설정**: 실체 관계 추출 & 페르소나 생성 & 에이전트 구성 주입
3. **시뮬레이션**: 이중 플랫폼 병렬 시뮬레이션 & 자동 예측 요구사항 파싱 & 동적 시계열 기억 업데이트
4. **보고서 생성**: 풍부한 툴셋을 가진 ReportAgent로 시뮬레이션 후 환경과 깊은 상호작용
5. **딥 인터랙션**: 시뮬레이션 월드의 모든 에이전트와 대화 & ReportAgent와 상호작용

## 🚀 빠른 시작

### 옵션 1: 소스 코드 배포 (권장)

#### 전제 조건

| 도구 | 버전 | 설명 | 설치 확인 |
|------|---------|-------------|-------------------|
| **Node.js** | 18+ | 프론트엔드 런타임, npm 포함 | `node -v` |
| **Python** | ≥3.11, ≤3.12 | 백엔드 런타임 | `python --version` |
| **uv** | 최신 | Python 패키지 매니저 | `uv --version` |

#### 1. 환경변수 구성

```bash
# 예제 구성 파일 복사
cp .env.example .env

# .env 파일 편집 및 필요한 API 키 입력
```

**필수 환경변수:**

```env
# LLM API 구성 (OpenAI SDK 형식의 모든 LLM API 지원)
# 권장: 알리바바 Qwen-plus 모델 via Bailian Platform: https://bailian.console.aliyun.com/
# 소비가 크므로 먼저 40회 미만의 시뮬레이션으로 시도
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Zep Cloud 구성
# 월간 무료 할당량으로 간단한 사용 충분: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 2. 의존성 설치

```bash
# 모든 의존성 일클릭 설치 (루트 + 프론트엔드 + 백엔드)
npm run setup:all
```

또는 단계별 설치:

```bash
# Node 의존성 설치 (루트 + 프론트엔드)
npm run setup

# Python 의존성 설치 (백엔드, 가상환경 자동생성)
npm run setup:backend
```

#### 3. 서비스 시작

```bash
# 프론트엔드와 백엔드 동시 시작 (프로젝트 루트에서 실행)
npm run dev
```

**서비스 URL:**
- 프론트엔드: `http://localhost:3000`
- 백엔드 API: `http://localhost:5001`

**개별 시작:**

```bash
npm run backend   # 백엔드만 시작
npm run frontend  # 프론트엔드만 시작
```

### 옵션 2: Docker 배포

```bash
# 1. 환경변수 구성 (소스 코드 배포와 동일)
cp .env.example .env

# 2. 이미지 풀 및 시작
docker compose up -d
```

기본적으로 루트 디렉토리의 `.env`를 읽고, 포트 `3000 (프론트엔드) / 5001 (백엔드)`를 매핑합니다

> `docker-compose.yml`에 주석으로 빠른 이미지 주소가 제공되어 있으니 필요시 교체하세요.
